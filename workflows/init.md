## Steps

- name: scan
  agent: null
  gate: auto
  rules: []
  context: []
  description: >
    Dispatcher scans project root directly (no agent needed):
    - Detect language/framework from marker files:
      DESCRIPTION -> R, package.json -> JS/TS, pyproject.toml/setup.py -> Python,
      Cargo.toml -> Rust, go.mod -> Go, pom.xml -> Java
    - Detect test framework:
      testthat -> R, pytest/unittest -> Python, jest/vitest -> JS,
      cargo test -> Rust, go test -> Go
    - Detect source file patterns from project structure
    - Detect existing CLAUDE.md and its contents
    - Detect existing .claude/ directory (agents, specs, hooks, rules)

- name: propose
  agent: null
  gate: structural
  rules: [knowledge]
  context: []
  description: >
    Present detected config to user for confirmation:
    - Language, test command, check command, source patterns, conventions
    - Which existing CLAUDE.md sections already satisfy harness requirements
    - What will be appended vs. what already exists
    - Any conflicts between existing content and harness expectations
    User must confirm before any changes are made.

- name: apply
  agent: null
  gate: structural
  rules: [knowledge]
  context: []
  description: >
    Append ## Harness Config section to CLAUDE.md.
    Never overwrite existing content — only append.
    If CLAUDE.md doesn't exist, create a minimal one with:
    - Project description placeholder
    - Harness Config section
    Show the user exactly what will be written before writing.
    Also ensure `.ewh-artifacts/` is in the project's .gitignore
    (add the line if .gitignore exists and doesn't already contain it;
    create .gitignore with that line if it doesn't exist).
